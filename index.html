Sobre Arborgram

Notación inspirada en qtree (gramática de corchetes simple para recursive-descent en JS) — con extensiones propias: AVM, multilínea, subíndices, tags de movimiento y recuadros de constituyente. Cuadro IPA armado sobre la grilla oficial de consonantes (lugar × modo de articulación) y vocales de la International Phonetic Association. Macros de griego, lógica/conjuntos y semántica siguiendo convenciones estándar de LaTeX (amsmath, amssymb, stmaryrd). Tipografías de sistema: Georgia o sans-serif (a elección, en Ajustes) para las etiquetas del árbol, system-ui para la interfaz.

Sintaxis
Estructura
[XP ... ]
nodo «XP» que domina lo que viene después
[N perro]
N domina la hoja «perro» (línea recta)
[SN el gato negro]
hoja de varias palabras → triángulo (constituyente sin analizar)
[SN <Bordemar>]
triángulo forzado aunque sea una sola palabra
[e] / [PRO]
nodo terminal solo — categorías vacías, huellas, etc.
[S This [ [V is] a tree ] ]
nodo sin etiqueta: sus hijos quedan pegados a la altura del padre
[A uno] [B dos] [C tres]
múltiples árboles yuxtapuestos separados por espacio
Notaciones de etiqueta
t_{i} / PRO_{i}
subíndice dentro de la etiqueta
\theta / \forall / \llbracket
macros de griego, lógica/conjuntos y semántica; también podés tipear el unicode directo. Ver teclado de símbolos arriba
ST'
las etiquetas pueden llevar apóstrofo, tildes, etc.
[Cº] / [T⁰]
etiqueta terminada en º/°/⁰¹²... → negrita sin cursiva aunque sea hoja (núcleos vacíos)
["n"P] / \i{n}P
cursiva parcial dentro de la etiqueta — útil para nP, aP, vP
[__Juan__]
doble guion bajo: cursiva forzada en todo el nodo, incluso si tiene hijos
\i{texto} · \b{texto} · \u{texto} · \s{texto}
estilos inline: cursiva, negrita, subrayado, tachado. Anidables: \b{\i{texto}}
[TP [DP Juan=1] [T' ... [DP Juan=1] ...]]
multidominancia: =ID al final marca nodo compartido. Primera aparición = nodo primario; segunda = segunda madre. Se dibuja una sola vez
Recuadro y tachado
[SN! el gato negro]
! al final de la etiqueta: recuadro punteado alrededor de todo el constituyente (el nodo y lo que domina)
el gato negro+
+ al final de una hoja: tachado (copia baja en cadenas de movimiento). el+gato+negro sin + final no tacha
el gato negro%1+!
%N, ! y + se pueden combinar en cualquier orden
Flechas de movimiento
qué%1 ... t%1
%N al final de la etiqueta: dos nodos con el mismo N quedan unidos por una flecha curva. Los chips →1 →2 →3 insertan el tag automáticamente en pares
%1.dotted / %1.solid / %1.dashed
estilo de línea para ese tag
%1.end / %1.start / %1.both / %1.none
punta de flecha para ese tag
%1.blue / %1.red / %1.#3a7bd5
color — paleta: red blue green purple orange teal gray, o #hex
%1.out30.in-20
ángulo de salida y entrada de la curva bezier en grados (positivo = derecha, negativo = izquierda)
%1.dip0.5
profundidad de la curva como factor del valor global (1 = default; dip2 = más profunda, dip0.2 = casi plana)
%1.lbl:Agree
etiqueta sobre la flecha, en itálica. Guión bajo = espacio: lbl:Internal_Merge. Aparece a la derecha por default; .left la pasa a la izquierda
%1.dotted.both.blue.out30.dip1.5.lbl:Move
todos los modificadores son combinables en cualquier orden
Multilínea
[N comprar|segunda línea]
| separa líneas dentro de la etiqueta de un nodo
Matrices de rasgos (AVM)
[V comprar|@CASO:nom;NUM:sg]
@ después del | abre una matriz AVM bajo la etiqueta; ; separa filas, : separa atributo y valor
[@TIPO:sn;CASO:nom]
sin texto antes del @: la matriz es toda la etiqueta del nodo
@AGR:{NUM:sg;PERS:3}
valor anidado: una matriz dentro de otra con { }
@REF:^1
^1 como valor → etiqueta de coindexación en caja
Structure
[XP ... ]
node dominating what follows
[N dog]
N dominates the leaf «dog» (straight line)
[NP the black cat]
multi-word leaf → triangle (unanalyzed constituent)
[NP <word>]
forced triangle even for a single word
[e] / [PRO]
single terminal node — empty categories, traces, etc.
[S This [ [V is] a tree ] ]
unlabeled node: its children attach at the parent row
[A one] [B two] [C three]
multiple juxtaposed trees separated by space
Label notation
t_{i} / PRO_{i}
subscript within label
\theta / \forall / \llbracket
Greek, logic/sets, semantics macros; unicode also works directly
TP'
labels can include apostrophes, accents, etc.
[Co] / [T⁰]
label ending in o/°/⁰¹²... → bold without italic even as a leaf (empty heads)
["n"P] / \i{n}P
partial italic within a label — useful for nP, aP, vP
[__John__]
double underscore: forced italic on the whole node, even with children
\i{t} · \b{t} · \u{t} · \s{t}
inline styles: italic, bold, underline, strikethrough. Nestable: \b{\i{t}}
[TP [DP John=1] [T' ... [DP John=1] ...]]
multidominance: =ID marks a shared node. First = primary; second = second mother. Drawn once
Box and strikethrough
[NP! the black cat]
! at end of label: dotted box around the whole constituent
the black cat+
+ at end of leaf: strikethrough (lower copy in movement chains)
the black cat%1+!
%N, ! and + combinable in any order
Movement arrows
what%1 ... t%1
%N at end: two nodes with the same N connected by a curved arrow
%1.dotted / %1.solid / %1.dashed
line style for that tag
%1.end / %1.start / %1.both / %1.none
arrowhead for that tag
%1.blue / %1.red / %1.#3a7bd5
color — palette: red blue green purple orange teal gray, or #hex
%1.out30.in-20
exit/entry angle of the bezier curve in degrees
%1.dip0.5
curve depth as a factor (1 = default; dip2 = deeper, dip0.2 = nearly flat)
%1.lbl:Agree
label on the arrow, italic. Underscore = space. .left moves it left
%1.dotted.both.blue.out30.dip1.5.lbl:Move
all modifiers combinable in any order
Multiline
[N buy|second line]
| separates lines within a node label
Feature matrices (AVM)
[V buy|@CASE:nom;NUM:sg]
@ after | opens an AVM; ; = rows, : = attr:value
[@TYPE:np;CASE:nom]
no text before @: the matrix is the entire label
@AGR:{NUM:sg;PERS:3}
nested value: a matrix inside another with { }
@REF:^1
^1 as value → coindexation tag in a box
